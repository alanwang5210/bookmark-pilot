Bookmark Pilot

Privacy Policy / 隐私政策

Last updated / 最后更新：2026-04-30

English

Bookmark Pilot is a Chrome extension focused on helping users search and manage bookmarks, folders, and tabs. This Privacy Policy explains what data the extension accesses, how that data is used, and when data may be sent to third-party services chosen by the user.

1. Data the extension accesses

  • Bookmarks and bookmark folders, so the extension can provide unified search and organization.
  • Open tab information, including tab title, URL, and window relationship, so tabs can be included in unified search results.
  • Extension settings and local state, such as language preference, AI settings, scope selection, and indexing status.
  • Optional page metadata, such as page title or meta description, only when the user enables enhanced AI analysis.

2. How data is used

  • To index and search bookmarks, folders, and tabs inside the extension.
  • To save user preferences locally on the user's device.
  • To generate optional AI labels and summaries when the user enables AI features.
  • To check whether selected bookmark or tab URLs appear to be dead links when the user starts dead-link scanning.

3. Local storage

Bookmark Pilot stores settings and extension state locally using chrome.storage.local and IndexedDB. This may include AI configuration entered by the user, saved scope selection, usage statistics used by the extension experience, and AI-generated labels or summaries created for bookmarks, folders, or tabs.

4. AI features and third-party services

AI features are optional and only run when the user enables them and provides an AI service endpoint and API key.

When AI analysis is used, Bookmark Pilot may send selected item metadata to the user-configured AI provider, including:

  • Item title
  • Item URL
  • Folder path
  • Entity type such as bookmark, folder, or tab
  • Optional page description metadata when enhanced AI analysis is enabled

The developer of Bookmark Pilot does not operate the user-selected AI endpoint and is not responsible for the privacy practices of that third-party AI provider.

5. Dead-link scanning

When the user runs dead-link scanning, the extension sends network requests to the URLs selected for checking in order to determine whether those URLs return errors such as 404, 410, DNS failure, or connection failure.

6. Public website analytics

This public privacy policy page uses Google Analytics to understand basic page traffic. Google may receive information such as page views, browser information, approximate location, and device information according to Google's own policies. Bookmark Pilot's Chrome extension pages do not load Google Analytics.

7. Data sharing

Bookmark Pilot does not sell user data. Data is not shared with the developer's own server, because the extension does not use a developer-operated backend service. Data is only transmitted externally when the user explicitly uses features that require network access, such as AI analysis, dead-link scanning, or visiting this public privacy policy page.

8. User control

  • Users can disable AI features at any time.
  • Users can clear AI-generated labels inside the extension.
  • Users can choose the AI scope and whether to include tabs.
  • Users can remove the extension to delete extension-managed local data from Chrome according to Chrome's extension data behavior.

9. Contact

For privacy questions about Bookmark Pilot, please contact the developer using the support contact information provided on the Chrome Web Store listing for this extension.

中文

Bookmark Pilot 是一款帮助用户统一搜索和管理书签、文件夹与标签页的 Chrome 扩展。 本隐私政策说明该扩展会访问哪些数据、这些数据如何被使用，以及在什么情况下数据可能会被发送到用户自行选择的第三方服务。

1. 扩展会访问哪些数据

  • 书签与书签文件夹：用于提供统一搜索与整理能力。
  • 打开的标签页信息：包括标题、URL 和所属窗口关系，用于将标签页纳入统一搜索结果。
  • 扩展设置与本地状态：例如语言偏好、AI 设置、生成范围选择和索引状态。
  • 可选的页面元数据：仅在用户开启增强 AI 分析时，可能读取页面标题或 meta description 等轻量信息。

2. 数据如何被使用

  • 用于在扩展内部建立索引并搜索书签、文件夹和标签页。
  • 用于在用户设备本地保存偏好设置。
  • 在用户主动启用 AI 功能时，用于生成可选的 AI 标签和一句话说明。
  • 在用户主动启动失效链接扫描时，用于检查指定书签或标签页 URL 是否失效。

3. 本地存储

Bookmark Pilot 使用 chrome.storage.local 和 IndexedDB 在用户设备本地保存扩展设置和状态。 其中可能包括用户填写的 AI 配置、保存的生成范围、扩展内部使用的使用状态，以及为书签、文件夹或标签页生成的 AI 标签与摘要。

4. AI 功能与第三方服务

AI 功能是可选功能，仅会在用户主动启用，并配置 AI 服务地址和 API Key 后运行。

当用户使用 AI 分析时，扩展可能会向用户自行配置的 AI 服务发送所选项目的以下信息：

  • 项目标题
  • 项目 URL
  • 文件夹路径
  • 项目类型，例如书签、文件夹或标签页
  • 若开启增强 AI 分析时，可能附带页面描述类元数据

Bookmark Pilot 的开发者不运营用户所配置的 AI 服务端点，也不对该第三方 AI 服务提供商的隐私做法负责。

5. 失效链接扫描

当用户执行失效链接扫描时，扩展会向被检查的 URL 发起网络请求，以判断这些地址是否返回 404、410、DNS 失败或连接失败等结果。

6. 公开网页分析

本公开隐私政策页面使用 Google Analytics 了解基础页面访问情况。Google 可能会根据其自身政策接收页面访问、 浏览器信息、大致位置和设备信息等数据。Bookmark Pilot 的 Chrome 扩展页面不会加载 Google Analytics。

7. 数据共享

Bookmark Pilot 不会出售用户数据。该扩展不使用开发者自建后端服务，因此数据不会被发送到开发者自己的服务器。 只有在用户主动使用需要联网的功能时，例如 AI 分析、失效链接扫描，或访问本公开隐私政策页面时，相关数据才会被发送到对应的目标地址。

8. 用户控制

  • 用户可以随时关闭 AI 功能。
  • 用户可以在扩展中清除 AI 生成的标签和摘要。
  • 用户可以自定义 AI 生成范围，并决定是否包含标签页。
  • 用户可以卸载扩展；扩展本地数据将按 Chrome 的扩展数据处理方式移除。

9. 联系方式

如有与 Bookmark Pilot 隐私相关的问题，请通过 Chrome Web Store 中该扩展条目提供的开发者联系信息与开发者联系。
